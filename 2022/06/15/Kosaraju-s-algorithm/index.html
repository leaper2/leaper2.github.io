Kosaraju's algorithm

Posted on 2022-06-15

The components-clasped graph of an original graph must be an acyclic graph in which the components act as super vertices.

  • Since the component graph is a DAG, we can topologically sort it. With topological ordering, the last component must have only incoming super edges pointed to it, no outgoing super edges. When we DFS with a vertex from the last component, we’ll traverse exhaustively and only traverse the vertices of the component. In the reverse topological order of the component graph, we can DFS and mark each component one by one.

  • Reverse post-order of the transposition of G(reverse of G), is not the same as the post-order of G.

      It may work for the graph you give, but not for all graphs. Consider a graph with 3 vertices, A, B, and C.
  A→B,C
  B→A
  Search on A, exploring B before C, B finishes first but is not in a sink.
  Also, note that for the Wikipedia article, the search is on incoming edges, not outgoing, which is why it doesn't need to explicitly transpose the graph.
  From <https://cs.stackexchange.com/questions/60503/reversing-topo-order-of-the-original-graph-instead-of-the-topo-order-of-the-tran>

    • Start from A, if you first visit C, then the post-order is C, B, A. When we DFS with this ordering, we would conclude that (B, A), (C) are two strongly connected component.

    • Start from A, if you first visit B, then the post-order is B, C, A. When we DFS with this ordering, we would conclude that (B, C, A) consist a strongly connected component.

    • However, if we use the transposition of G, say G’, starting from A, we have a reverse post-order of C, A, B, which will give the correct result. Actually, with G’, no matter which vertex you choose to start, the resulting reverse post-order would give us the correct strongly connected components.

    • The principle reason is the first vertex of the post-order of the original graph may or may not be a sink vertex(which has no outgoing edges). It differs from the case of the topological ordering problem due to, in topological ordering there is no cycle, whereas strongly connected component problem is generally concerned with cycles.
      If we rule out cycles in the first place, then the graph becomes A→B,C without B→A, then there has no difference for visiting either B or C firstly in terms of topological order.

    • Why the transposition of G could surface the sink vertex

      • Because the cycles in the original graph would be preserved after transposition, only the sink would become prominent.
      • If the sink vertex in the original graph have parallel not-sink vertices with the same head vertex, transposing G would surface the sink.
      • Post-order of original graph works good when there is no parallel not-sink vertices together with sink vertex.
